# M2 P3.5 — Hardening Roadmap (pre-P4)

> Integrated plan merging the `/forge` design critique (`forge/2026-06-16-0007.md`, 12 findings),
> the `/inspect --vision` health+vision report (`inspections/2026-06-16-0031.md`, 1 P0 / 12 P1 /
> 18 P2 / 25 P3 + 73 vision proposals), and `docs/MILESTONES.md`. Read-only analyses; this file is
> the execution plan. Nothing here has been applied.

**Goal:** Close the one P0 and the cluster of P1 contract-violations, then harden the compiler/VM
seams that M2 P4 (classes/enums/match + arena) would otherwise inherit and multiply — *before*
starting P4.

**Unifying thesis (why these findings cluster):** Phorge declares two contracts it does not yet
fully enforce.
1. **Backend parity** — `run` ≡ `runvm`, byte-identical *including error/exit behaviour*. The
   differential harness is the project's spine, but `agree()` compares only Ok-path stdout, so the
   parity contract has a structural hole. The P0 `Op::Neg` divergence lives precisely in that hole.
2. **No-crash-on-input (EV-7)** — malformed/adversarial `.phg` must exit 1 with a clean error,
   never abort. Today the VM caps recursion but the interpreter and the recursive-descent
   parser/checker do not → two SIGABRT (exit 134) classes.

Every P0/P1 is a facet of one of these two contracts, or of the **duplication** that lets parity
drift (arith/compare implemented twice). **P4 is the forcing function:** forge repeatedly notes each
seam gap "gets worse at P4" (bytecode index validation, stack-balance asserts, `num_ty` partiality,
the contextless error channel, the un-interned constant pool feeding the future GC root set). Doing
the hardening as a named **P3.5 wave before P4** is cheaper than doing it after P4 widens every
surface.

**Architecture of the work:** five waves, ordered so each wave's regression net exists before the
next wave needs it. W0 builds the oracle and stops the crashes; W1 removes the divergence mechanism;
W2 adds diagnostics/limits; W3 measures and tells the truth in docs; W4 is milestone-aligned and
explicitly deferred to land *with* P4/P5, not before.

**Tech stack:** Rust edition 2021, std-only, no external crates. Differential harness in
`tests/differential.rs` is the canonical regression oracle. Op↔vm.rs match is exhaustive (adding an
`Op` variant requires the `vm.rs` arm in the same commit — see memory `op-variant-match-coupling`).

## Execution decisions log
- [2026-06-16] DONE (Task 2.1): three execution-time refinements to the roadmap sketch, all chosen at
  implementation time and verified:
  1. **`Diagnostic` carries `line`/`col`, not the full `Span`.** The sketch said `{stage,span,message}`,
     but `token::Span` also has `start`/`len` byte offsets that *no* error renderer consumes, and every
     construction site already had a `line`/`col` in hand. line/col is also sufficient for the future
     `--json`/LSP range (`0` = unknown). Embedding the full `Span` would force byte offsets on every
     site for no reader.
  2. **Runtime render is `at {line}`, not `{func}:{line}`.** Dropping `func` keeps `Diagnostic` uniform
     across all five stages (a `func` field would be dead weight on four of them); the source line is the
     user-meaningful locator and `func` is an internal index. Licensed by "Open decisions: resolve at
     execution time".
  3. **Full replacement, no type aliases.** Field names (`message`/`line`/`col`) already matched the old
     structs, so migration was: drop struct + add `stage:` to each literal + retype signatures — a
     mechanical, compiler-verified sweep. Leaves zero alias debt for 2.2+/LSP.
  - **Known limitation, deferred (NOT a regression):** a runtime fault *inside string interpolation*
    (`"{1 / z}"`) reports **line 1**, because `parser::split_interpolation` re-lexes the inner expression
    with a fresh lexer that resets to line 1 — interpolated sub-expressions lose the outer source
    position. This is a pre-existing parser/position-accuracy issue, orthogonal to 2.1 (which faithfully
    wires whatever `Chunk.lines` holds), and adjacent to the spec's parked "Diagnostics column-semantics"
    item. Fix = thread the outer span into the interpolation sub-lexer; defer to the LSP/diagnostics layer.

  existing program/test recurses past ~12, and deeper programs SIGABRT the interpreter today anyway,
  so this is parity gained, not capability lost. Value/stack-size move into `Limits` at Task 2.2.
