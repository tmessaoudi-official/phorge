# M2 P3.5 ‚Äî Hardening Roadmap (pre-P4)

> Integrated plan merging the `/forge` design critique (`forge/2026-06-16-0007.md`, 12 findings),
> the `/inspect --vision` health+vision report (`inspections/2026-06-16-0031.md`, 1 P0 / 12 P1 /
> 18 P2 / 25 P3 + 73 vision proposals), and `docs/MILESTONES.md`. Read-only analyses; this file is
> the execution plan. Nothing here has been applied.

**Goal:** Close the one P0 and the cluster of P1 contract-violations, then harden the compiler/VM
seams that M2 P4 (classes/enums/match + arena) would otherwise inherit and multiply ‚Äî *before*
starting P4.

**Unifying thesis (why these findings cluster):** Phorge declares two contracts it does not yet
fully enforce.
1. **Backend parity** ‚Äî `run` ‚â° `runvm`, byte-identical *including error/exit behaviour*. The
   differential harness is the project's spine, but `agree()` compares only Ok-path stdout, so the
   parity contract has a structural hole. The P0 `Op::Neg` divergence lives precisely in that hole.
2. **No-crash-on-input (EV-7)** ‚Äî malformed/adversarial `.phg` must exit 1 with a clean error,
   never abort. Today the VM caps recursion but the interpreter and the recursive-descent
   parser/checker do not ‚Üí two SIGABRT (exit 134) classes.

Every P0/P1 is a facet of one of these two contracts, or of the **duplication** that lets parity
drift (arith/compare implemented twice). **P4 is the forcing function:** forge repeatedly notes each
seam gap "gets worse at P4" (bytecode index validation, stack-balance asserts, `num_ty` partiality,
the contextless error channel, the un-interned constant pool feeding the future GC root set). Doing
the hardening as a named **P3.5 wave before P4** is cheaper than doing it after P4 widens every
surface.

**Architecture of the work:** five waves, ordered so each wave's regression net exists before the
next wave needs it. W0 builds the oracle and stops the crashes; W1 removes the divergence mechanism;
W2 adds diagnostics/limits; W3 measures and tells the truth in docs; W4 is milestone-aligned and
explicitly deferred to land *with* P4/P5, not before.

**Tech stack:** Rust edition 2021, std-only, no external crates. Differential harness in
`tests/differential.rs` is the canonical regression oracle. Op‚Üîvm.rs match is exhaustive (adding an
`Op` variant requires the `vm.rs` arm in the same commit ‚Äî see memory `op-variant-match-coupling`).

## Wave 4 ‚Äî Milestone-aligned (do WITH the milestone, NOT before)

These are real, but doing them now is premature; each is pinned to where it pays off.
- **Value handle/`Rc`-ification ‚Üí land WITH P4 arena** (`VH-P2`, FRG data-model). `Rc` can't handle
  the cycles the P5 GC must; do it *as* the arena + GC work, not as a standalone pass.
- **Tracing/mark-sweep GC = P5** (already the milestone).
- **Full typed-AST threading** (the "real fix" for 1.4) ‚Äî do when P4's type needs force it; the cheap
  `num_ty` extension covers P4 entry.
- **Property/generative testing** of lexer‚Üíparser‚Üídifferential pipeline (`VD`) ‚Äî high value once the
  `agree_err` oracle (0.1) exists to be the property's assertion.
- **`phorge disasm` + trace/timings tooling** (`VB`) ‚Äî Op-additive, couples to vm.rs (memory
  `op-variant-match-coupling`); fold into P4 when Op churn happens anyway.
- **Then M2 P4 proper** ‚Äî classes/enums/`match` + arena, now landing on hardened seams.

## Execution decisions log
- [2026-06-16] AGREED (Task 0.3): recursion limit is a **single shared `MAX_CALL_DEPTH`**
  (in `value.rs`) enforced by *both* backends ‚Äî the interpreter's new depth counter and the VM
  (replacing `MAX_FRAMES = 64*1024`). Rationale: separate limits would open a parity-divergence
  band (interpreter errors at L while the VM succeeds up to 65536) ‚Äî the exact hole-class this wave
  closes. Measured evidence: the interpreter recurses on the *native* Rust stack and SIGABRTs
  between 850‚Äì900 simple frames on the default 12.2 MB stack (~14 KB/frame, debug), so literally
  mirroring 65536 is impossible. To decouple the logical limit from Rust's frame size, `interpret()`
  runs on a dedicated 256 MB-stack thread (`thread::Builder::spawn_scoped`), making
  `MAX_CALL_DEPTH = 4096` reachable with >4√ó native margin. VM max-recursion drops 65536‚Üí4096; no
  existing program/test recurses past ~12, and deeper programs SIGABRT the interpreter today anyway,
  so this is parity gained, not capability lost. Value/stack-size move into `Limits` at Task 2.2.
- [2026-06-16] AGREED: Adopt rustfmt. The committed codebase was never `cargo fmt`'d
  (compact manual style, 198 hunks across all 19 files; no `rustfmt.toml`). Resolution: a
  one-time isolated `style: apply rustfmt baseline` commit lands *before* Wave 0, making
  `cargo fmt --check` truly clean ‚Äî so the autonomy gate, the Task 0.5 pre-commit hook, and
  the Task 3.2 `rustfmt.toml` all become real rather than aspirational. The single-line
  struct-variant style is not preservable via stable rustfmt knobs, so it is intentionally
  given up.
