# M-mut.6 — shared-mutable instance fields `o.f = e` (+ cycle-safe `eq_val`)

> Slice 6 of the mutation milestone (`docs/specs/2026-06-21-mutation-milestone-design.md` §4) — the
> first *handle*-semantics mutation: instances become shared-mutable, so `o.f = e` is visible through
> every binding to the same instance (PHP/Java object semantics, F2). Crosses the GC boundary, but
> **no collector is built** (Fork-3 = defer to per-process reclaim; cycles leak at process exit, which
> is fine for a run-once CLI). Autonomy: directive `f5f2912`.

**STATUS: COMPLETE.** Byte-identical run≡runvm≡real PHP 8.6; 654 tests, clippy+fmt clean. VM stays
11.74× on the object workload (handle change is borrow-only on `GetField`; `GetLocal` unchanged).

## Design (locked by §1–§7 of the milestone spec)

- **Representation: `struct Instance { class: String, fields: RefCell<HashMap<String, Value>> }`, held in
  `Rc<Instance>`** (Alternative B — the immutable `class` stays outside the cell; only the field map is
  interior-mutable). Cloning a `Value::Instance` is a refcount bump on a *shared* cell ⇒ handle
  semantics. Field reads `inst.fields.borrow().get(..).cloned()`; writes `inst.fields.borrow_mut()
  .insert(..)`. Borrows are never held across a re-entrant `eval`/`run` (read clones out and drops the
  borrow; SetField evaluates the value *before* `borrow_mut`).
- **F4 (P0 prereq): `eq_val` cycle-safe.** New private `eq_val_rec(other, &mut visited)` where `visited`
  is a `Vec<(*const Instance, *const Instance)>`; the `Instance` arm short-circuits a re-encountered
  pair to `true` (co-inductive bisimulation — the standard correct cyclic equality), preventing the
  native-stack overflow a cycle would otherwise cause. Memoization (no pop) is sound: a *false* pair
  short-circuits the whole comparison, so a stale-true memo can never be observed. Threaded through
  every recursive arm (List/Map/Set/Enum/Instance); only `Instance` consults/extends `visited`.
- **`mutable` fields.** New `Modifier::Mutable` (`parse_modifiers` accepts the existing `mutable`
  keyword in field + ctor-param position). Immutable by default; only a `mutable` field may be the
  target of `o.f = e` (`E-ASSIGN-IMMUTABLE`). Checker `ClassInfo` gains `mutable_fields: HashSet<String>`.
- **One new `Op::SetField(name_idx)`** — the three coupled matches: `vm::exec_op` (pop value, pop
  instance, `borrow_mut().insert`, push nothing), `compiler::stack_effect` = −2, `chunk::validate`
  joins the `GetField | CallMethod` name-pool-bound arm.
- **`Stmt::Assign` target extended to `Expr::Member`** (`o.f = e`, `this.f = e`). Checker
  `check_field_assign`: resolve object → class → field must exist (`E-ASSIGN-UNKNOWN`) + be `mutable`
  (`E-ASSIGN-IMMUTABLE`) + value assignable to the field type (`E-ASSIGN-TYPE`). `this.f` resolves via
  `cur_class`. Nested places (`a.b.c`, `this.f[i]`) still `E-ASSIGN-TARGET` (deferred).
- **Backends.** Interpreter `Stmt::Assign` Member arm (eval object, eval value, `borrow_mut().insert`);
  compiler Member arm (`expr(object); expr(value); SetField(idx)`); transpiler lhs `Expr::Member =>
  emit_expr(target)` ⇒ `$o->f = $e;`. The ctor body may now set a `mutable` field via `this.f = e`
  (works on the VM because the synthetic ctor shares one `Rc` between `this_slot` and the returned
  instance — a `borrow_mut` is visible in the result).
- **Deferred (KNOWN_ISSUES):** cycle collector (Fork-3); `===` identity equality (optional, §6); nested
  place-stores; `readonly`/`final` PHP emission (immutable-by-default already prevents the write).

## Scope NOT changing observable behavior of existing programs

The `Rc<Instance>` → `Rc<Instance{RefCell fields}>` switch is behavior-preserving for all immutable
programs (reads identical). All 24 examples + every existing test stay byte-identical; only the new
mutation surface is added. (Verified by the full gate at the end.)
